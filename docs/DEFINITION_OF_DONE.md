# DEFINITION_OF_DONE.md

Ένα task θεωρείται ολοκληρωμένο όταν:

- έχει υλοποιηθεί σωστά
- είναι μέσα στο συμφωνημένο scope
- βρίσκεται στο σωστό αρχείο
- έχει καθαρή ονοματοδοσία
- έχει docstrings όπου χρειάζεται
- δεν σπάει άλλα κομμάτια
- έχει ελεγχθεί από τον team lead
- μπορεί να γίνει merge στο test branch
